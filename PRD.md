# PRD — 학원 출석체크 서비스

**문서 버전** v0.4 | **작성일** 2026.04.28 | **작성자** 원장 (PM)

> v0.4 변경사항: 클래스(반) 개념 도입 — 학생을 반에 배정하고 매주 수업 요일을 지정한다.
> 결석 판정과 미등원 집계가 "학생이 속한 반의 수업 요일"을 기준으로 동작한다.
> 출석체크 자체는 요일과 무관하게 항상 허용 (보강 수업 지원).

---

## 1. 서비스 개요

### 배경 및 목적
학원 운영 중 수기 출석 관리의 불편함을 해소하고, 학생 등원/하원 즉시 부모님께 자동으로 알림을 발송하여 학부모 안심 및 원장 업무 효율화를 목적으로 한다.

### 사용자

| 구분 | 설명 | 접근 기기 |
|---|---|---|
| 학생 | 학원 도착/퇴원 시 출석 체크 | 학원 내 태블릿 (고정) |
| 학부모 | 등원/하원 알림 수신 | 본인 휴대폰 (카카오톡) |
| 원장 | 출석 현황 관리 | 모바일 / PC |

---

## 2. 사용자 시나리오

### 학생 등원/하원 체크 흐름

1. 학생이 학원 도착 후 태블릿 화면에서 **[등원 / 하원]** 버튼 선택
2. 부모님 전화번호 **뒷자리 4자리** 입력 (숫자 키패드)
3. 일치하는 학생이 **1명**이면 → 즉시 처리
4. 일치하는 학생이 **2명 이상**이면 → 이름 목록 표시 → 본인 선택
5. 처리 완료 → 부모님께 **카카오 알림톡** 자동 발송
6. 태블릿 화면은 완료 메시지 표시 후 **초기 화면으로 자동 복귀**

### 메시지 예시

```
등원: [학원명] 홍길동 학생이 오후 3:05에 등원했습니다.
하원: [학원명] 홍길동 학생이 오후 5:30에 하원했습니다.
```

## 3. 기능 요구사항

### 3-1. 태블릿 출석 화면 (학생용)

- **[등원 / 하원]** 선택 버튼이 메인 화면에 표시
- 선택 후 숫자 키패드로 4자리 입력 (터치 친화적 UI)
- 번호 중복 시 이름 목록 선택 화면으로 전환
- 등록되지 않은 번호 입력 시 → "등록되지 않은 번호입니다" 안내 후 재입력
- 처리 완료 후 자동 초기화 (다음 학생 대기 상태)
- **쿨타임**: 동일 학생, 동일 타입(등원→등원 / 하원→하원) 기준 마지막 처리로부터 **5분 이내 재입력 차단**
  - 등원 후 하원은 쿨타임 없이 허용
- 당일 동일 학생 중복 등원/하원 허용 (오전/오후 수업 모두 기록)
- **클래스 요일과 무관하게 항상 처리** — 보강 수업 케이스를 위해 수업 없는 날에도 등원/하원 + 알림톡이 정상 발송된다

### 3-2. 알림 발송

- 처리 완료 즉시 **카카오 알림톡** 발송 (목표: 5초 이내)
- 발송 실패 시 **최대 3회 자동 재시도** (Exponential Backoff)
  - 1차 실패 → 1분 후 재시도
  - 2차 실패 → 5분 후 재시도
  - 3차 실패 → 15분 후 재시도
- 3회 모두 실패한 경우에만 관리자 페이지에 실패 내역 표시

### 3-3. 관리자 페이지 (원장용)

| 기능 | 상세 |
|---|---|
| 대시보드 | 오늘 등원/하원/미등원/발송 실패 카운트 + 최근 활동 + 미등원 알림 배너 (등원/미등원 분모는 "오늘 수업이 있는 활성 학생") |
| 출석 기록 조회 | 기간(7일/30일/전체) · 등원/하원 · 학생 이름 필터, 메모 인라인 편집 |
| 결석 관리 | 학생별 클래스 요일 합집합에 해당하는 (학생 × 날짜) 중 등원 기록 없는 행 + **보강 메모 인라인 기록**(검색·기간 필터 + 결석 인원/메모 미등록 지표 카드) |
| 학생·학부모 관리 | 등록 / 수정 / 소프트 삭제(`is_active=false`), 학부모 N명 관리, 소속 반 칩 표시 |
| **클래스 관리** | 반 등록·수정·삭제, 매주 수업 요일(0=일~6=토) 지정, 학생 다중 배정 (검색 + 체크박스) |
| 발송 실패 내역 | 3회 재시도 후에도 실패한 알림톡 목록 확인 |

### 3-4. 클래스(반) 운영 규칙

- 클래스는 `(이름, 수업 요일 배열)`로 정의된다. 수업이 매주 같은 요일에 일정하게 잡힌다는 가정.
- 학생 ↔ 클래스는 **다대다** — 한 학생이 영어반·수학반 등 여러 반에 동시에 속할 수 있다.
- **결석 판정**: 학생이 속한 모든 반의 수업 요일을 합집합으로 본다. 어느 한 반이라도 수업 있는 날에 등원 기록이 없으면 결석 후보가 된다.
- **클래스 미배정 학생**: 결석 정의 자체가 없으므로 결석 집계와 미등원 카드 분모에서 자동 제외된다 (출석/알림은 정상 처리).
- **휴강일**: 어느 반도 수업이 없는 요일(weekdays에 미포함)은 결석 후보로 잡지 않는다.

## 5. 엣지케이스 및 예외 처리

| 상황 | 처리 방식 |
|---|---|
| 뒷자리 4자리 중복 학생 2명 이상 | 이름 목록 표시 후 선택 |
| 등록되지 않은 번호 입력 | "등록되지 않은 번호입니다" 안내 후 재입력 |
| 동일 타입 5분 이내 재입력 | 쿨타임 안내 메시지 표시 후 차단 |
| 등원 후 하원 체크 | 쿨타임 없이 정상 처리 |
| 당일 동일 학생 재등원/재하원 (5분 이후) | 중복 허용, 모두 기록 |
| 카카오 알림톡 발송 실패 | 최대 3회 자동 재시도, 이후 관리자 페이지에 표시 |
| 태블릿 장시간 미사용 | 초기 화면 유지 (자동 복귀 상태) |
| 수업 없는 날 (학생의 반 weekdays에 미포함) 등원 시도 | 보강 수업으로 보고 정상 처리 + 알림톡 발송 |
| 클래스 미배정 학생 출석 | 정상 처리 + 알림톡 발송. 단 결석 집계·미등원 카드에서는 분모에서 제외 |
| 휴강일 (어느 반도 수업 없음) | 결석 후보 0건, 미등원 분모 0명 → 카드는 모두 0으로 표시 |
